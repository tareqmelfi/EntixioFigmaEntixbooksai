## 1. نظرة عامة على المنصة

**Entix Books** هي منصة محاسبة مالية سحابية متكاملة (SaaS) مبنية بـ React + TypeScript + Tailwind CSS v4، تستهدف:
- **السوق السعودي**: متوافقة مع هيئة الزكاة والضريبة والجمارك (ZATCA)، الفاتورة الإلكترونية (Phase 2)، GOSI، نظام مدد لرواتب الموظفين.
- **السوق الأمريكي**: متوافقة مع GAAP، 1099/W-9، Sales Tax متعدد الولايات.

### الخصائص الأساسية للنظام
- ✅ دعم كامل ثنائي اللغة (عربي/إنجليزي) مع RTL/LTR ديناميكي.
- ✅ خطوط: **Noto Sans Arabic** (للعربية) و **Inter** (للإنجليزية).
- ✅ نظام ألوان محدد ومعتمد:
  - **Navy** `#0B1A47` / `#0B1B49` → لصالحنا / مدفوع / إيجابي.
  - **Blue** `#1276E3` → اللون الرئيسي للتفاعل.
  - **Teal** `#349FC4` / `#179FC5` → علينا / مستحق.
  - **Red** `#EF4444` → الخسائر فقط (لا يستخدم للمستحقات).
  - **بادج علينا** خلفية `#FEF3C7` (أصفر/ذهبي) - نص داكن.
  - **بادج لصالحنا** خلفية `#DCFCE7` (أخضر فاتح) - نص داكن.
- ✅ معيار UX-11 لعرض العملة: الرقم أولاً ثم رمز العملة (مثال: `1,250.00 SAR`).
- ✅ منع استخدام الأرقام العربية الهندية تماماً - استخدام الأرقام اللاتينية (0-9) فقط.
- ✅ تصميم Responsive كامل (Mobile / Tablet / Desktop).
- ✅ Dark Mode مدعوم بالكامل.

## 12. ⭐ صفحة الفواتير الذكية - التصميم الكامل

### 12.1 الفلسفة
صفحة الفواتير `/app/invoices` ليست مجرد جدول بيانات، بل **مركز قيادة ذكي** يستقبل الفواتير من مصادر متعددة ويعالجها تلقائياً قبل عرضها للمراجعة.

### 12.2 طرق إدخال الفاتورة (Multi-Input)

#### أ) الإدخال اليدوي التقليدي
- زر **"+ فاتورة جديدة"** يفتح نافذة Dialog كاملة.
- نموذج كامل: العميل، التاريخ، البنود، الضرائب، الخصومات، الإجمالي.
- بحث ذكي عن المنتجات (`item-search-input.tsx`).
- بحث ذكي عن العملاء (`contact-search-input.tsx`).
- اختيار حساب محاسبي (`account-select.tsx`).
- اختيار معدل ضريبي (`tax-rate-select.tsx`).

#### ب) النسخ واللصق الذكي (Smart Paste) 🆕
**منطقة Drop Zone في أعلى الصفحة:**
- يمكن لصق (Ctrl+V / Cmd+V) أي محتوى:
  - **نص منسوخ** من بريد إلكتروني/واتساب/SMS.
  - **جدول من Excel/Google Sheets** → يُحلل تلقائياً ويملأ البنود.
  - **صورة** من الحافظة (Screenshot) → يدخل في خط OCR.
  - **PDF** ملصوق → يُستخرج النص ويُحلل.
- حدث `onPaste` يلتقط `clipboardData` ويوجهها للمعالج المناسب.
- مؤشر بصري حي: "تم اكتشاف صورة - جاري الاستخراج..." / "تم اكتشاف جدول - جاري التحليل...".

#### ج) السحب والإفلات (Drag & Drop)
- منطقة استقبال للملفات: PDF, JPG, PNG, HEIC.
- دعم رفع متعدد (Batch Upload) لمعالجة عدة فواتير دفعة واحدة.

#### د) الكاميرا (Mobile-First)
- زر "التقاط بالكاميرا" يستخدم `<input type="file" capture="environment">`.
- ضبط تلقائي للزاوية والإضاءة قبل OCR.

#### هـ) البريد الإلكتروني
- لكل شركة بريد فريد: `invoices-{tenantId}@inbox.entixbooks.com`.
- أي فاتورة ترسل لهذا البريد تظهر في الصندوق الوارد للمعالجة.

#### و) WhatsApp Business API
- إعادة توجيه فواتير من WhatsApp مباشرة.

### 12.3 خط معالجة OCR الذكي (Pipeline)

```
[المدخل] → [التحضير] → [OCR] → [الاستخراج] → [التحقق] → [الاقتراح] → [المراجعة] → [الترحيل]
```

#### المرحلة 1: التحضير (Pre-processing)
- تحويل HEIC → JPEG.
- ضبط التباين والإضاءة.
- تصحيح الميل (Deskew).
- إزالة الضوضاء.
- تقسيم الصفحات (PDF متعدد).

#### المرحلة 2: OCR (Text Extraction)
- **محركان**: Tesseract.js (محلي) + Google Vision / Azure / AWS Textract (سحابي).
- دعم اللغة العربية والإنجليزية في نفس المستند.
- استخراج النص مع إحداثيات (bounding boxes) لكل كلمة.

#### المرحلة 3: الاستخراج الذكي (Field Extraction)
نموذج NLP/LLM مخصص يستخرج:
- **رقم الفاتورة** (Invoice Number).
- **التاريخ** (مع تحويل التقويم الهجري ↔ الميلادي).
- **اسم المورد** + **الرقم الضريبي** (مطابقة مع قاعدة بيانات جهات الاتصال).
- **البنود**: وصف، كمية، سعر، إجمالي.
- **الضريبة** (VAT 15%).
- **الإجمالي قبل وبعد الضريبة**.
- **رمز QR للفاتورة الإلكترونية** (ZATCA Phase 2): يتم فك التشفير TLV لاستخراج البيانات الموثقة.

#### المرحلة 4: التحقق (Validation)
- مطابقة المجاميع رياضياً (Subtotal + VAT = Total).
- التحقق من الرقم الضريبي عبر API ZATCA.
- كشف التكرار (Duplicate Detection) بناءً على رقم الفاتورة + المورد.
- تنبيه عند اختلاف العملة عن الافتراضية.

#### المرحلة 5: الاقتراح الذكي (Smart Suggestions)
- اقتراح **حساب محاسبي** بناءً على نمط المورد السابق (Machine Learning).
- اقتراح **مركز التكلفة / المشروع** بناءً على البنود.
- اقتراح **معدل ضريبي**.
- ثقة (Confidence Score) لكل حقل: <70% أصفر، <90% برتقالي، ≥90% أخضر.

#### المرحلة 6: المراجعة (Human-in-the-Loop)
- شاشة مقسمة: صورة الفاتورة الأصلية على اليمين، الحقول المستخرجة على اليسار.
- النقر على حقل يبرز موقعه في الصورة (highlight bounding box).
- تعديل أي حقل يُغذي النموذج للتعلم المستمر.

#### المرحلة 7: الترحيل (Posting)
- زر "اعتماد وترحيل" → ينشئ:
  - قيد محاسبي تلقائي (Journal Entry).
  - حركة على شجرة الحسابات.
  - تحديث الذمم/المخزون.
  - ربط المرفق (الصورة/PDF) بالقيد.

### 12.5 حالات الفاتورة (Lifecycle States)
- 🟡 **مسودة** - لم تُرسل بعد.
- 🔵 **مرسلة** - أُرسلت للعميل.
- 🟢 **مدفوعة** (Navy `#0B1B49`).
- 🟠 **مدفوعة جزئياً**.
- 🔴 **متأخرة** (Red `#EF4444` للخسائر/التأخر الكبير).
- ⚫ **ملغاة**.
- ✅ **مرحلة لـ ZATCA** (مع رقم Clearance).

### 12.6 العمليات الجماعية (Bulk Actions)
- اختيار متعدد + إجراء جماعي.
- إرسال جماعي للعملاء.
- ترحيل جماعي.
- تصدير دفعة كملف ZIP.
- معالجة OCR متوازية.

### 12.7 التكامل مع ZATCA (Phase 2)
- توليد رمز QR (TLV → Base64).
- توقيع رقمي (Digital Signature) بشهادة من ZATCA.
- إرسال للـ Clearance API قبل الإرسال للعميل.
- تخزين UUID + Hash لكل فاتورة.
- XML بصيغة UBL 2.1.

### 12.8 الأمان والامتثال
- تشفير الملفات المرفقة (AES-256).
- سجل تدقيق كامل (Audit Log) لكل تعديل.
- صلاحيات دقيقة (RBAC): إنشاء، تعديل، اعتماد، إلغاء.
- نسخ احتياطية تلقائية.

## 13. البنية التقنية (Tech Stack)

### Frontend
- **React 18** + **TypeScript**.
- **Tailwind CSS v4** (بدون tailwind.config.js).
- **shadcn/ui** components.
- **React Router** للتنقل.
- **recharts** للرسوم البيانية.

### مكتبات مثبتة ⚙️
- `react-to-print` - الطباعة المباشرة.
- `jspdf` + `html2canvas` - تصدير PDF.
- `react-textarea-autosize` - مربعات نص متجاوبة.
- `slick-carousel` + `react-slick` - عروض متحركة.
- `lucide-react` - الأيقونات.
- `motion/react` - الأنيميشن.
- `react-dnd` - السحب والإفلات.

### State & Data
- `auth-store.ts` - مصادقة المستخدم.
- `contacts-store.tsx` - مخزن جهات الاتصال.
- `data-service.ts` - طبقة الخدمات.
- `LanguageContext.tsx` - السياق اللغوي.

### المكونات المشتركة
- `app-header.tsx` - الهيدر العلوي.
- `app-sidebar.tsx` - القائمة الجانبية.
- `auth-guard.tsx` - حماية المسارات.
- `error-boundary.tsx` - معالجة الأخطاء.
- `useDirectionClasses.tsx` - hook للاتجاه (RTL/LTR).
- `quote-print-template.tsx` - قالب طباعة عروض الأسعار.
